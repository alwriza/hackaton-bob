# OPENWORK — ТЕХНИЧЕСКИЙ ОТЧЁТ ДЛЯ ОЦЕНЩИКА
> Биржа безопасного фриланса для подростков
> Stack: Next.js 15, TypeScript, Supabase (Auth + PostgreSQL + Realtime), Tailwind CSS 4

---

## 1. ОБЩАЯ КОНЦЕПЦИЯ

OPENwork — это MVP платформы для безопасного фриланса между подростками (13–17 лет) и клиентами.
Ключевые принципы:
- **Нет фиксированных ролей**: каждый пользователь может одновременно быть покупателем и продавцом
- **Escrow-защита**: деньги блокируются на платформе и поступают исполнителю только после подтверждения
- **Родительский контроль**: родитель может мониторить сделки ребёнка, блокировать транзакции и устанавливать лимиты
- **Safety AI**: система фильтрации сообщений, блокирующая обмен контактами

### 2.3. РЕГИСТРАЦИЯ (`/register`)

**Поля формы:**
- Имя пользователя (text)
- Email (email)
- Пароль (password)
- **Выбор роли** (карточки-переключатели):
  - "Пользователь" — может работать и заказывать
  - "Родитель" — контроль безопасности ребёнка
- **Email родителя** — появляется только если выбрана роль "Пользователь"; используется для привязки к родительскому аккаунту

**Логика:**
1. `supabase.auth.signUp()` — создаёт auth-пользователя
2. `supabase.from('profiles').insert()` — создаёт строку профиля с полями: `name`, `role`, `parent_email`, `trust_score=100`, `balance=0`, `completed_jobs=0`
3. `supabase.auth.signInWithPassword()` — немедленно авторизует пользователя
4. Редирект на главную страницу

**Обработка ошибок:** отображается inline сообщение об ошибке в форме.

**Автовосстановление профиля:** если пользователь зарегистрировался но профиль не создался — `auth-context` автоматически создаёт его при следующем входе.

**Поля формы:**

3. Graceful fallback не блокирует создание услуги

**Кнопки:**
- "Отмена" — возврат назад через `router.back()`
- "Опубликовать услугу" — отправка формы с лоадером

### 2.9. СТРАНИЦА ТРАНЗАКЦИИ / ЧАТ (`/transaction/[id]`)

**Самая функциональная страница платформы**

**Заголовок:**
- Кнопка назад

- Название услуги
- Статусный бейдж (АКТИВНА / ЗАМОРОЖЕНА / ЗАВЕРШЕНА / СПОР) — меняет цвет
- Кнопка **"SOS"** — красная (скрыта если сделка завершена или заморожена)

**Чат (левая область):**
- Полноэкранный чат с пузырями сообщений
  - Мои сообщения — синие, выравнивание вправо
  - Чужие сообщения — белые, выравнивание влево
  - Системные сообщения — серый бейдж по центру
- **Realtime**: новые сообщения появляются без перезагрузки (Supabase Realtime через postgres_changes)
- **Safety AI фильтр**: блокирует сообщения содержащие: `whatsapp`, `telegram`, `телеграм`, `ватсап`, `+7`, `номер`, `instagram` — показывает предупреждение
- **Поле ввода**: textarea + кнопка отправки, блокируется при FROZEN/COMPLETED
- **Кнопка Paperclip (📎)** — загрузка файла:
  - Открывает диалог выбора файла (images, pdf, doc, zip, rar)
  - Отправляет системное сообщение в чат: "📎 Файл загружен: [имя] ([размер] KB)"
- **Статусные баннеры** поверх чата:
  - Красный — при FROZEN ("Чат заморожен...")
  - Зелёный — при COMPLETED ("Сделка успешно завершена!")
  - Жёлтый — при DISPUTED ("Открыт спор...")

**Боковая панель (правая):**
- **Карточка Escrow**: сумма к выплате в ₸, текущий статус цветом
- **Кнопки действий (зависят от роли и статуса):**
  - Покупатель при ACTIVE:
    - ✅ **"Подтвердить выполнение"** — меняет статус на COMPLETED + отправляет системное сообщение в чат
    - ⚠️ **"Открыть спор"** — открывает модальное окно
  - Продавец при ACTIVE: информационный блок "Ожидайте подтверждения заказчика"
  - При COMPLETED: информация о выплате средств
  - При FROZEN: информация о заморозке
- **Safety Card** — напоминания о правилах безопасности

**Модальное окно SOS:**
- Большой значок предупреждения
- Кнопка **"ДА, МНЕ НУЖНА ПОМОЩЬ"** → статус FROZEN + системное сообщение в чат
- Кнопка "Отмена"

**Модальное окно "Открыть спор":**
- Поле textarea для описания причины спора
- Кнопка "Открыть спор" (активна только если причина введена) → статус DISPUTED + системное сообщение с причиной
- Кнопка "Отмена"

## 3. СИСТЕМА АУТЕНТИФИКАЦИИ

**Провайдер:** Supabase Auth

**Файл:** `lib/auth-context.tsx`

**Поведение:**
- При запуске: проверяет текущую сессию через `supabase.auth.getSession()`
- Подписывается на события: `SIGNED_IN`, `INITIAL_SESSION`, `SIGNED_OUT`
- Загружает профиль из таблицы `profiles` по `user.id`
- **Автосоздание профиля:** если пользователь авторизован, но профиль не найден (PGRST116) — автоматически создаёт базовый профиль
- **Обработка lock-конфликтов:** при открытии нескольких вкладок браузера возникает `navigator.lock` конфликт — обрабатывается gracefully (молча игнорируется)

**Данные пользователя (User):**
- `id`, `name`, `email`, `role` (USER / PARENT)
- `trust_score` (0-100%)
- `balance` (в условных единицах ₸)
- `completed_jobs` (число выполненных сделок)
- `avatar` (URL изображения)
- `parent_email` (для привязки к родителю)

## 5. РЕАЛИЗОВАННЫЕ СИСТЕМЫ БЕЗОПАСНОСТИ

### 5.1 Escrow System
- При заказе создаётся транзакция со статусом `ACTIVE` — деньги считаются "заморожены"
- Статус меняется на `COMPLETED` только когда покупатель нажимает "Подтвердить выполнение"
- Исполнитель не может самостоятельно завершить сделку

### 5.2 Safety AI (Chat Filter)
- Блокирует отправку сообщений содержащих:
  - мессенджеры: `whatsapp`, `telegram`, `телеграм`, `ватсап`
  - контакты: `+7`, `номер`, `instagram`
- При попытке — показывает предупреждение, сообщение НЕ отправляется

### 5.3 Panic Button (SOS)
- Красная кнопка SOS в шапке чата
- Открывает модальное окно с предупреждением
- При подтверждении:
  1. Статус транзакции → `FROZEN`
  2. В чат отправляется системное сообщение
  3. Поле ввода блокируется

### 5.4 Dispute System
- Кнопка "Открыть спор" у покупателя при активной сделке
- Требует ввода причины (текстовое поле)
- При подтверждении:
  1. Статус → `DISPUTED`
  2. В чат отправляется системное сообщение с причиной

### 5.5 Parent Control
- Родитель видит все транзакции привязанных детей
- Одна кнопка мгновенно замораживает ВСЕ активные сделки ребёнка
- Настройка дневных лимитов (UI реализован, сохранение — alert-уведомление)
- Привязка через email при регистрации ребёнка

### 5.6 Proof of Work
- В чате есть кнопка 📎 (Paperclip) для загрузки файла
- При выборе файла — в чат отправляется системное сообщение с именем и размером файла
- Все сообщения хранятся в БД с timestamp

## 6. UX / ДИЗАЙН

- **Цветовая схема:** белый фон, синий (primary) для основных действий, зелёный (success) для подтверждения, красный (destructive) для опасных действий, жёлтый (warning) для рейтинга/спора
- **Типографика:** рубленые шрифты, bold/black начертания, uppercase tracking-widest для лейблов
- **Анимации:** `animate-in fade-in`, `slide-in-from-bottom`, `zoom-in-95` при появлении страниц и модалов
- **Адаптивность:** колонки перестраиваются на мобильных (1 → 2 → 3 колонки для каталога)
- **Загрузочные состояния:** спиннеры Loader2 на каждом асинхронном запросе
- **Пустые состояния:** кастомные иллюстрации с описанием и CTA кнопками
- **Модальные окна:** полноэкранный backdrop + blur, центрированный белый контейнер с закруглёнными углами
- **Карточки:** `rounded-[40px]`, `shadow-premium`, hover-анимации с scale и shadow
